# 🍽️ 취향 저격 매치메이커 (Taste Matchmaker)

사용자의 미세한 취향, 위치, 방문 목적을 분석하여 최적의 맛집, 빵집, 카페를 큐레이션해주는 **하이브리드 추천 웹 서비스**입니다.

## 🌟 프로젝트 개요

일반적인 평점 기반 식당 추천 서비스와 달리, 사용자의 기분, 날씨, 동행자, 텍스처(식감) 선호도 등 세밀한 "취향 조건"을 다각도로 질문하고 분석합니다.
단순 객관식 퀴즈 형태를 넘어, 특정 분기점에서는 사용자가 직접 주관식으로 지역명이나 키워드를 검색할 수 있는 **객관식 + 주관식 하이브리드 검색 시스템**을 갖추고 있습니다.

### 주요 기능 (Features)

1. **라멘 몬스터 (Ramen Matcher) 🍜**
   - 육수 베이스, 면의 굵기, 선호 토핑(차슈, 타마고 등)을 물어보며 정통 일본 라멘 10가지(쇼유, 시오, 돈코츠, 츠케멘, 마제소바 등) 중 딱 맞는 장르와 해당 장르의 유명 식당을 추천합니다.
   
2. **비 오는 날 추천 (Rainy Day) ☔**
   - 전, 국밥, 칼국수 등 비 오는 날 당기는 음식과 세부 재료를 분석해 최적의 한식당을 매칭합니다.

3. **타코 매니아 (Taco Lover) 🌮**
   - 정통 멕시칸 스타일부터 미국식 텍스멕스(Tex-Mex)까지, 고기 종류와 조리 방식을 세밀하게 선택해 타코 맛집을 찾습니다.

4. **빵지순례 (Bakery Recommend) 🥐**
   - 빵의 종류나 가고 싶은 동네를 직접 **텍스트로 검색**하면, 리뷰 교차 검증 및 타 매장 비교 대조군이 명확한 전국 핫플 빵집 21곳 중에서 추천해줍니다. 웨이팅 정보, 품절 정보가 별도로 분리되어 관리됩니다.

5. **카페 매칭 테스트 (Cafe Curation) ☕**
   - 방문 목적(드라이브, 카공, 커피맛, 인스타 감성)을 객관식으로 선택한 뒤 동네를 검색하면, **대형카페 유무, 콘센트 유무, 디카페인, 반려동물 동반 가능 여부** 등의 세부 운영 필터가 자동 적용되어 결과가 렌더링됩니다.

## 🛠️ 기술 스택 (Tech Stack)

- **Backend**: Java 17, Spring Boot 3.3.0, Spring Data JPA
- **Database**: H2 Database (In-Memory), 추후 PostgreSQL 확장 용이하도록 JPA 연동
- **Frontend**: Vanilla HTML/CSS/JavaScript (외부 프레임워크 없는 경량화 구조)
- **Data Pipeline**: Python (추출 스크립트) → TSV 포맷 → Java `CommandLineRunner` (Seeder)로 DB 자동 주입

## 🏛️ 아키텍처 및 데이터 모델링 (Architecture)

본 프로젝트는 도메인의 성격(음식점 vs 베이커리 vs 카페)에 따라 정보의 스키마가 완전히 다르다는 점에 착안해, **다형성(Polymorphism) 및 1:1 관계를 활용한 3중 엔티티 설계**를 채택했습니다.

### 1. 동적 객관식 퀴즈 엔진
- `Theme` 1 : N `Question` 1 : N `Option`
- 각 `Option`은 다음 질문(`nextQuestionId`)으로 연결되는 포인터를 가져, 트리 구조의 퀴즈 시나리오를 구성합니다.
- `Option`의 `InputType` 속성을 통해 특정 버튼 클릭 시 **텍스트 입력창**으로 자연스럽게 전환됩니다.

### 2. 베이커리 / 카페 도메인 분할 설계
- **핵심 정보 (`Bakery`, `Cafe`)**: 매장명, 분위기, 긍정/부정 요약 리뷰, 리뷰 출처 링크(리스트)
- **운영 실용 정보 (`BakeryOperation`, `CafeOperation`)**: 웨이팅, 픽업 예약, 대형카페 유무, 반려동물 동반, 콘센트 유무 등을 1:1로 매핑하여 핵심 데이터의 오염 방지
- **타 매장 비교 지표 (`BakeryComparison`, `CafeComparison`)**: "(A)카페는 (B)카페보다 식감이 바삭하다" 와 같이 대조군 식당과의 비교 우위를 별도로 1:N 저장

## 🚀 실행 방법 (How to Run)

### 1. 로컬 환경에서 바로 실행
```bash
# 레포지토리 클론 후 프로젝트 최상단 디렉토리에서
./mvnw spring-boot:run
```

### 2. 브라우저 접속
- `http://localhost:8080` 으로 접속하면 프론트엔드 UI를 확인할 수 있습니다.

### 3. 데이터 파이프라인 (Data Generation)
- 파이썬 스크립트를 통해 새로운 베이커리/카페 데이터를 크롤링하거나 수작업으로 채워넣어 갱신할 수 있습니다.
```bash
python generate_bakeries.py
python generate_cafes.py
# 생성된 .tsv 파일은 /src/main/resources 폴더에 자동 저장되며,
# 스프링 구동 시 Seeder가 이를 읽어 DB에 주입합니다.
```
